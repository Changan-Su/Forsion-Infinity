### 追加记录 - 上传文件 500 错误

#### 现象

- 上传文件时报 `500 Internal Server Error`，错误信息：`column "isHtmlBundle" of relation "drive_file" does not exist`。
- `pnpm migrate` 显示 "No migrations are pending"，但数据库表结构仍缺少列。

#### 原因

- 代码模型 `DriveFile.ts` 已包含 `isHtmlBundle` 和 `htmlBundlePath` 字段，但迁移系统未检测到需要添加这些列的迁移文件。
- 可能是新功能添加了字段但迁移文件尚未创建或未正确执行。

#### 处理方式

手动执行 SQL 添加缺失列：

```sql
ALTER TABLE drive_file 
ADD COLUMN IF NOT EXISTS "isHtmlBundle" boolean NOT NULL DEFAULT false,
ADD COLUMN IF NOT EXISTS "htmlBundlePath" character varying(1024);
```

#### 结论

- 当迁移系统显示无待执行迁移但代码模型与数据库结构不一致时，需要手动检查并补齐缺失的列。
- 建议检查是否有遗漏的迁移文件，或等待上游补全迁移。

### 追加记录 - 设置页面加载失败

#### 现象

- 访问设置页面时出现 `TypeError: Cannot read properties of undefined (reading 'persisted')` 错误。
- 错误位置：`storage.ts:11:61`，在访问 `navigator.storage.persisted()` 时。

#### 原因

- `navigator.storage` API 在某些环境下不可用（如非 HTTPS、某些浏览器版本）。
- 代码直接调用 `navigator.storage.persisted()` 未做安全检查。

#### 处理方式

在 `packages/frontend/src/utility/storage.ts` 中添加安全检查：

```typescript
// 修复前
export const storagePersisted = ref(await navigator.storage.persisted());

// 修复后
export const storagePersisted = ref(await (navigator.storage?.persisted?.() ?? Promise.resolve(false)).catch(() => false));
```

同时在 `enableStoragePersistence()` 函数中添加 `navigator.storage?.persist` 的存在性检查。

#### 结论

- 使用浏览器新 API 时需要添加兼容性检查，避免在不可用环境下崩溃。
- 修复后需要重新构建前端或等待开发模式自动重载。

### 实现方式

#### 1. 前端修改

**文件：`packages/frontend/src/components/MkPostForm.vue`**

- **工具栏按钮**（第 99 行）：
  - 添加了新按钮 `<i class="ti ti-file-code"></i>`，显示为代码文件图标
  - 按钮位置：在"从云盘选择"和"投票"按钮之间
  - 提示文本：`i18n.ts.attachHtmlBundle`

- **上传函数**（第 507-522 行）：
  ```typescript
  function chooseHtmlBundle(ev: PointerEvent) {
    if (props.mock) return;
    
    os.chooseFileFromPc({ 
      multiple: false,
      accept: 'application/zip,.zip'
    }).then(selectedFiles => {
      if (selectedFiles.length === 0) return;
      
      // 显示说明对话框
      os.confirm({
        type: 'info',
        title: i18n.ts.attachHtmlBundle,
        text: i18n.ts.htmlBundleDescription,
      }).then(({ canceled }) => {
        if (!canceled) {
          uploader.addFiles(selectedFiles);
        }
      });
    });
  }
  ```

#### 2. 国际化翻译

**添加的翻译键：**

- `attachHtmlBundle`: 按钮提示文本

**各语言版本：**

- **英文** (`locales/en-US.yml`):
  - `attachHtmlBundle`: "Attach HTML Bundle (ZIP)"

#### 3. 功能特点

1. **文件类型限制**：只接受 `.zip` 文件（`accept: 'application/zip,.zip'`）
2. **单文件上传**：`multiple: false`，每次只能上传一个 HTML Bundle
3. **用户友好**：上传前显示说明对话框，告知用户 ZIP 包的要求
4. **复用现有上传器**：使用 `uploader.addFiles()`，与普通文件上传流程一致

#### 4. 后端支持（已存在）

后端已有完整的 HTML Bundle 处理机制：

- **自动识别**：ZIP 文件（`mime: application/zip`）自动识别为 HTML Bundle
- **解压验证**：`HtmlBundleService.extractHtmlBundle()` 解压并验证
- **安全检查**：
  - 文件数量限制（`HTML_BUNDLE_MAX_FILES: 100`）
  - 大小限制（`HTML_BUNDLE_MAX_SIZE: 10MB`）
  - 禁止的扩展名检查（`HTML_BUNDLE_FORBIDDEN_EXTENSIONS`）
  - 路径遍历防护
  - 必须包含 `index.html`
- **存储**：解压到 `files/html-bundles/{fileId}/`
- **提供**：通过 `/files/{fileId}/html-bundle/index.html` 访问

#### 5. 前端显示（已存在）

- **组件**：`MkMediaHtmlBundle.vue` 在帖子中展示 HTML Bundle
- **预览状态**：折叠状态显示文件名和"点击查看"提示
- **展开状态**：在 `<iframe>` 中加载 HTML Bundle 内容
- **安全性**：`sandbox="allow-scripts allow-same-origin"`

### 使用方法

1. 准备 HTML Bundle：
   - 创建包含 `index.html`（必须）、CSS、JS 的文件夹
   - 打包为 ZIP 格式：`zip -r bundle.zip *`

2. 发帖时上传：
   - 点击发帖工具栏的 📄 图标（文件代码图标）
   - 选择 ZIP 文件
   - 阅读说明后确认上传
   - ZIP 会自动识别为 HTML Bundle

3. 查看效果：
   - 发布后，帖子中会显示 HTML Bundle 预览
   - 点击预览展开，在 iframe 中显示交互内容

### 技术要点

- **前端限制 ZIP 类型**：避免用户误选其他文件
- **后端验证内容**：确保 ZIP 包含有效的 HTML Bundle
- **沙箱隔离**：iframe 的 `sandbox` 属性保证安全性
- **说明引导**：对话框提示帮助用户理解要求
